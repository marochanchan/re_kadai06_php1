# ①課題名
クローゼット管理アプリ（DB版 Ver.3）

## ②課題内容（どんな作品か）
- PHPとMySQLを使用して、洋服を登録・管理できるクローゼット管理アプリを作成しました。
- 洋服の「登録・一覧表示・編集・削除（CRUD）」に対応しています。
- 画像アップロード機能を実装し、洋服を写真付きで管理できます。
- シーズン検索、カテゴリ検索、並び替え、お気に入り機能を追加し、一覧画面の使いやすさを向上させました。
- ログイン機能・ユーザー管理機能を実装し、管理者と一般ユーザーで操作できる内容を変更しています。

## ⑤工夫した点・こだわった点
- 前回制作したクローゼット管理アプリをベースに、認証機能を追加して実際のサービスに近づけました。
- ログイン機能を実装し、管理者・一般ユーザーで表示内容や操作権限を切り替えています。
- 管理者のみ洋服の登録・編集・削除・お気に入り登録・ユーザー追加ができるようにしました。
- 一般ユーザーは一覧閲覧や検索のみ可能とし、権限管理を意識した設計にしました。
- シーズン検索・カテゴリ検索・並び替え・お気に入り表示を組み合わせても検索条件が維持されるようGETパラメータを工夫しました。
- カテゴリボタンはデータベースから自動生成し、登録件数も表示することで管理しやすくしました。
- 登録数・お気に入り数を表示するサマリーカードやカード型UIを作成し、ECサイトのような見た目を意識しました。
- ヘッダーやボタンデザインも統一し、操作しやすい画面になるようCSSを調整しました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- ログイン認証やセッション管理、管理者と一般ユーザーで表示・操作を切り替える権限制御の考え方を理解するのが難しかったです。
- 検索機能やお気に入り機能とログイン機能を組み合わせても正常に動作するよう処理を整理することに苦労しました。

次回以降挑戦したいこと

- 着用回数・着用履歴の記録
- クローゼット総額の表示
- おすすめコーデの提案機能（天気APIと連携したコーデ提案機能とか？）
