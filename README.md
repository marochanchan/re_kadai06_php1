# ①課題名
クローゼット管理アプリ（DB版）

## ②課題内容（どんな作品か）
- PHPとMySQLを使用して、洋服を登録・管理できるクローゼット管理アプリを作成しました。
- 洋服の「登録・一覧表示・編集・削除（CRUD）」に対応しています。
- 前回はCSVファイルでデータを管理していましたが、今回はMySQLデータベースへ移行し、データ管理をDB化しました。
- 画像アップロード機能を実装し、洋服を写真付きで管理できます。

## ⑤工夫した点・こだわった点
- 前回のCSV管理からMySQLデータベース管理へ移行し、PDOを用いたデータ登録・取得・更新・削除を実装しました。
- funcs.phpを作成し、データベース接続や共通処理を関数化してコードを整理しました。
- htmlspecialchars()を利用したh()関数を使用し、画面表示時のXSS対策を行いました。
- 画像アップロード機能を実装し、同じ画像名でも上書きされないように保存時に現在時刻を付与してファイル名が重複しないよう工夫しました。
- シーズン・カテゴリによる絞り込み検索機能を実装しました。
- GROUP BYを使用してカテゴリごとの登録枚数を集計し、一覧画面に表示する機能を追加しました。
- CSSでカード形式の一覧表示やフォームデザインを整え、見やすいUIを意識しました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- CSVで管理していたデータをデータベースへ置き換える際に、各ファイルの処理を一つずつ変更する必要があり、データの流れを理解するのに苦労しました。
- SQL文（SELECT、INSERT、UPDATE、DELETE）やPDOの使い方に慣れるまで時間がかかりました。
- 今後はログイン機能を追加し、ユーザーごとにクローゼットを管理できるようにしたいです。
- また、ブランド別やシーズン別の円グラフなど、データベースの集計機能を活かした可視化にも挑戦したいです。

## ⑦フリー項目（感想、シェアしたいこと等なんでも）
- 前回作成したCSV版をベースに、データ管理をデータベースへ置き換えることで、CRUD処理の流れやデータベースを利用するメリットを理解することができました。
- funcs.phpで処理を共通化したことで、コードの見通しが良くなり、保守しやすい書き方を学ぶことができました。
- PHPとHTMLが複数ファイルに分かれているため最初は処理を追うのが大変でしたが、一つずつ動作を確認しながら修正することで理解が深まりました。
