# ①課題名
クローゼット管理アプリ（DB版 Ver.2）

## ②課題内容（どんな作品か）
- PHPとMySQLを使用して、洋服を登録・管理できるクローゼット管理アプリを作成しました。
- 洋服の「登録・一覧表示・編集・削除（CRUD）」に対応しています。
- 画像アップロード機能を実装し、洋服を写真付きで管理できます。
- シーズン検索に加え、カテゴリボタンによる絞り込みや並び替え機能を追加し、実際のECサイトを意識したUIへ改善しました。

## ⑤工夫した点・こだわった点
- 前回作成したDB版をベースに、一覧画面の使いやすさを改善する機能を追加しました。
- シーズンでの絞り込みに加え、カテゴリをボタン形式で表示し、ワンクリックでカテゴリ検索ができるようにしました。
- カテゴリボタンはデータベースから自動生成されるため、新しいカテゴリを登録するとボタンも自動で表示されます（専用アイコンを表示したい場合のみアイコン設定を追加します。）
- 各カテゴリの登録枚数も同時に表示し、データベースの集計機能を活用しました。
- 新しい順・古い順で表示順を切り替えられるようにし、利用シーンに応じて一覧を見やすくしました。
- 一覧画面と登録画面を相互に移動できる導線を追加し、操作しやすいUIを意識しました。
- お気に入り機能を追加し、カード右上のハートアイコンからお気に入り登録・解除ができるようにしました。また、お気に入りのみを表示するフィルターも実装しました。
- CSSでカードUIやカテゴリボタンをデザインし、ECサイトのような見た目を意識しました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- データベースの内容を利用してカテゴリボタンを自動生成する処理では、SQLのGROUP BYとPHPの繰り返し処理を組み合わせる考え方を理解するのが難しかったです。
- 検索条件（シーズン・カテゴリ・お気に入り・並び順）を組み合わせても条件が維持されるよう、GETパラメータの受け渡しを整理する部分に苦労しました。
- 今後はログイン機能を追加し、ユーザーごとにクローゼットを管理できるようにしたいです。
- また、着用回数の記録や着用履歴、クローゼット総額、おすすめコーデ表示など、実際のクローゼット管理アプリらしい機能にも挑戦したいです。
